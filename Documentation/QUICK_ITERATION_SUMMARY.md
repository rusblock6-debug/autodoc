# СТАРОЕ СОДЕРЖИМОЕ (СПРАВОЧНО)

# Краткая сводка итераций: Guidde-подобные улучшения

## 🎯 Главная цель
Улучшить существующую систему, НЕ переписывая на AI генерацию видео.

---

## Итерации (2-3 дня работы)

### ✅ Итерация 0: Подготовка (1-2 часа)
**Что делаем:**
- Создаем ветку `feature/guidde-improvements`
- Делаем бэкапы БД
- Проверяем что Whisper, LLM, TTS работают
- НЕ устанавливаем тяжелые библиотеки (diffusers, etc.)

**Результат:** Готовы к безопасной разработке

---

### ✅ Итерация 1: Улучшение AI текста/аудио (3-4 часа)
**Что делаем:**
- **Whisper:** Добавляем word-level timestamps, фильтруем мусорные слова
- **LLM:** Улучшаем промпты, добавляем контекст предыдущих шагов
- **Генерация заголовков:** Автоматические заголовки для шагов
- **TTS:** Добавляем SSML разметку, выбор голосов

**Файлы:** `app/services/ai_service.py`, `app/services/tts_service.py`

**Результат:** Транскрипция точнее, описания качественнее, озвучка естественнее

---

### ✅ Итерация 2: Улучшение FFmpeg обработки (4-5 часов)
**Что делаем:**
- **Step Detector:** Умная группировка кликов, удаление пауз
- **Screenshot Service:** Автоматический crop, улучшение качества
- **Video Processor:** Плавный зум на клики, transitions между шагами
- **Aligner:** Интеллектуальная синхронизация аудио-видео

**Файлы:** 
- `app/services/step_detector.py`
- `app/services/screenshot_service.py`
- `app/services/video_processor.py`
- `app/services/aligner.py`

**Результат:** Видео нарезается точно, паузы удаляются, зум плавный

---

### ✅ Итерация 3: "Магическое редактирование" (3-4 часа)
**Что делаем:**
- **Версионирование:** История изменений шагов
- **Magic Edit API:** Пользователь меняет текст → TTS регенерируется
- **Frontend:** Inline редактирование с автообновлением
- **Фоновая пересборка:** Видео обновляется автоматически

**Файлы:**
- `app/models.py` (добавить версионирование)
- `app/api/steps.py` (новый endpoint `/magic-edit`)
- `frontend/src/pages/StepEditor.jsx` (inline editing)
- `app/celery_tasks.py` (фоновая пересборка)

**Результат:** Ключевая фича Guidde - редактирование текста обновляет все

---

### ✅ Итерация 4: Удаленное развертывание (2-3 часа)
**Что делаем:**
- **Extension:** Настройка удаленного API URL, chunked upload
- **Backend:** CORS для production, rate limiting, HTTPS
- **Docker:** Production конфигурация с nginx
- **Документация:** Инструкции по развертыванию

**Файлы:**
- `extension/popup.js`, `extension/background.js`
- `app/main.py` (CORS, security)
- `docker-compose.prod.yml` (новый)
- `Documentation/REMOTE_DEPLOYMENT.md` (новый)

**Результат:** Приложение работает на удаленном сервере

---

### ✅ Итерация 5: Финал (2-3 часа)
**Что делаем:**
- **Субтитры:** Автоматическая генерация SRT
- **Экспорт:** PDF, PowerPoint, интерактивный HTML
- **UI/UX:** Поиск, фильтры, drag & drop
- **Тестирование:** End-to-end, нагрузочное
- **Документация:** Обновление всех гайдов

**Результат:** Готово к релизу v2.0

## 💰 Оценка ресурсов

### Время:
- **Минимум:** 15 часов (если все идет гладко)
- **Реалистично:** 18-20 часов (с отладкой)
- **Максимум:** 21 час (с проблемами)

### Кредиты (tokens):
- **Итерация 0:** ~5-10k tokens
- **Итерация 1:** ~20-30k tokens
- **Итерация 2:** ~30-40k tokens
- **Итерация 3:** ~20-30k tokens
- **Итерация 4:** ~15-20k tokens
- **Итерация 5:** ~10-15k tokens
- **ИТОГО:** ~100-145k tokens

**Доступно:** 140k tokens
**Нужно:** ~120k tokens
**Запас:** 20k tokens ✅

### GPU требования:
- **Whisper:** 2-4GB VRAM (tiny/base модель)
- **LLM:** 4-6GB VRAM (Qwen 2.5 7B Q4)
- **TTS:** CPU (Edge TTS) или 1-2GB VRAM
- **FFmpeg:** CPU (не требует GPU)
- **ИТОГО:** 6-8GB VRAM достаточно
