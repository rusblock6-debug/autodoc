# План развития НИР-Документ

## Архитектура системы

```
┌─────────────────┐
│ Chrome Extension│  ← Пользователь записывает действия
└────────┬────────┘
         │ 1. Захват экрана + клики
         ↓
┌─────────────────────────────────────────────────────┐
│              Backend (FastAPI)                       │
│                                                      │
│  POST /api/v1/sessions/upload                       │
│  ├─ Сохраняет скриншоты в data/screenshots/        │
│  ├─ Создает запись в БД (recording_sessions)       │
│  └─ Запускает Celery задачу                        │
└──────────────────┬──────────────────────────────────┘
                   │
                   ↓
┌─────────────────────────────────────────────────────┐
│           Celery Worker (фоновая обработка)         │
│                                                      │
│  process_session_task()                             │
│  ├─ 2. Анализ скриншотов через Vision AI           │
│  │    (OpenRouter: qwen/qwen3.6-plus:free)         │
│  │    • Отправляет скриншот + координаты клика     │
│  │    • Получает текст: "Нажмите на кнопку X"      │
│  │                                                  │
│  ├─ 3. Создает шаги (guide_steps)                  │
│  │    • step_number, instruction_text, screenshot  │
│  │                                                  │
│  └─ 4. Создает гайд (guides)                       │
│       • Связывает все шаги вместе                  │
└─────────────────────────────────────────────────────┘
```

## Поток данных

```
1. ЗАПИСЬ (Extension)
   ┌──────────────────────────┐
   │ Пользователь кликает     │
   │ ↓                        │
   │ Screenshot + координаты  │
   │ ↓                        │
   │ Сохраняется в extension  │
   └──────────────────────────┘

2. ЗАГРУЗКА (Backend API)
   ┌──────────────────────────┐
   │ POST /sessions/upload    │
   │ ↓                        │
   │ Сохранить скриншоты      │
   │ ↓                        │
   │ Создать session в БД     │
   │ ↓                        │
   │ Запустить Celery задачу  │
   └──────────────────────────┘

3. ОБРАБОТКА (Celery Worker)
   ┌──────────────────────────┐
   │ Для каждого клика:       │
   │ ↓                        │
   │ Vision AI анализ         │
   │ (скриншот + координаты)  │
   │ ↓                        │
   │ Получить инструкцию      │
   │ ↓                        │
   │ Создать step в БД        │
   └──────────────────────────┘

4. ПРОСМОТР (Frontend)
   ┌──────────────────────────┐
   │ GET /guides              │
   │ ↓                        │
   │ Показать список гайдов   │
   │ ↓                        │
   │ GET /guides/{id}/steps   │
   │ ↓                        │
   │ Показать шаги с картинками│
   └──────────────────────────┘
```

## Инструменты

**Запись:**
- Chrome Extension (background.js, content.js, popup.js)
- Захват экрана через `getDisplayMedia()`
- Отслеживание кликов через DOM events

**Backend:**
- FastAPI (Python) - REST API
- PostgreSQL - хранение гайдов/шагов
- Redis - очередь задач

**Обработка:**
- Celery Worker - фоновая обработка
- OpenRouter API - Vision AI анализ
- Model: `qwen/qwen3.6-plus:free`

**Фронтенд:**
- React + Vite
- Показывает гайды и шаги
