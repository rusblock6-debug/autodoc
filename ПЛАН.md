# 📋 План реализации функции записи голоса

## 🎯 Цель

Автоматическая запись голоса пользователя через расширение → расшифровка → создание готовых шагов с текстом.

---

## 🔄 Полный конвейер работы

### 1️⃣ Запись через расширение

**Файл:** `extension/background.js`

**Что записывается:**
- ✅ Клики (координаты, элементы)
- ✅ Скриншоты (PNG файлы)
- ✅ **Аудио с микрофона** (WebM файл) ← НОВОЕ!

**Процесс:**
```javascript
1. Пользователь нажимает "Начать запись"
2. Запрашивается доступ к микрофону
3. Запись аудио начинается через MediaRecorder
4. При каждом клике:
   - Сохраняются координаты
   - Делается скриншот вкладки
   - Аудио пишется непрерывно
5. Пользователь нажимает "Стоп"
6. Всё отправляется на сервер одной посылкой
```

**Важные нюансы:**
- ⚠️ Если микрофон недоступен - запись работает БЕЗ аудио
- ⚠️ Аудио сохраняется как один файл (не по сегментам)
- ⚠️ Формат: WebM (кодек Opus)

```python
@router.post("/upload")
async def upload_session(
    audio: Optional[UploadFile] = File(None),  # ← Принимает аудио
    ...
):
    # 1. Генерируем UUID сессии
    session_uuid = str(uuid4())

    if audio and audio.filename:

            audio.filename,

        )
        # Путь: /data/audio/{uuid}/recording.webm
    
    # 3. Создаём запись в БД
    session = RecordingSession(
        uuid=session_uuid,
        audio_path=audio_path or "",  # ← Сохраняем путь
        ...
    )
    
    # 4. Создаём гайд и шаги
    guide = Guide(...)
    for i, click in enumerate(clicks_data):
        step = GuideStep(
            guide_id=guide.id,
            step_number=i + 1,
            click_timestamp=click['timestamp'],
            # Если аудио есть - будет заменено позже
            # Если аудио нет - так и останется
            normalized_text=f"Шаг {i+1}: Нажмите на элемент {click.get('element') or 'unknown'}",
            ...
        )
    
    # 5. Запускаем обработку в фоне!
    if audio_path:
        from app.celery_tasks import ai_transcription
        celery_task = ai_transcription.delay(
            guide_id=guide.id,
            audio_key=audio_path,
            language="ru"
        )
        logger.info(f"Started transcription: {celery_task.id}")
    else:
        # Аудио нет - шаги уже созданы с текстом по умолчанию
        guide.status = GuideStatus.READY
        logger.info("No audio - using default step descriptions")
    
    return {"success": True, "guide_id": guide.uuid}
```

**Важные нюансы:**
- ✅ Обработка запускается АСИНХРОННО (Celery)
- ✅ API сразу возвращает ответ (не ждёт обработки)
- ✅ Если аудио нет - обработка пропускается
- ✅ Статус гайда: DRAFT (черновик)

### 4️⃣ Celery: Транскрипция (Whisper)

**Файл:** `workers/ai_runner.py`

**Задача:**
```python
def transcribe_audio(payload: dict) -> dict:
    audio_path = payload['audio_key']  # Один файл!
    
    # 1. Загружаем Whisper модель (1 раз!)
    import whisper
    model = whisper.load_model("medium")
    
    # 2. Транскрибируем ВСЁ аудио целиком
    result = model.transcribe(
        audio_path,
        language="ru",
        verbose=False
    )
    
    # Результат Whisper:
    {
        "text": "Полный текст всей записи",
        "segments": [
            {
                "text": "Ну тут надо нажать вот сюда...",
                "start": 1.5,  # секунды
                "end": 3.2,
                "confidence": 0.95
            },
            ...
        ]
    }
    
    return {
        "success": True,
        "transcription": result,
        "duration": result.get('duration', 0)
    }
```

**Время работы:**
- ⏱ ~1-2 минуты на 5 минут аудио (CPU)
- ⏱ Модель загружается 1 раз (~5 сек)

**Важные нюансы:**
- ✅ Whisper вызывается **ТОЛЬКО 1 РАЗ** для всего аудио
- ✅ НЕ вызывается для каждого скриншота/шага
- ✅ Работает в фоне, не блокирует интерфейс
- ✅ После обработки модель остаётся в памяти

### 5️⃣ Celery: Нормализация + Анализ скриншотов (Qwen)

**Файл:** `workers/ai_runner.py`

**Обработка:**
```python
from app.services.ai_service import LLMWrapper
from PIL import Image

llm = LLMWrapper()

normalized_steps = []

for i, segment in enumerate(transcription['segments']):
    raw_text = segment['text']  # "Ну тут надо нажать..."
    
    # Берём соответствующий скриншот
    screenshot_path = f"screenshots/{uuid}/screenshot_{i}.png"
    screenshot = Image.open(screenshot_path)
    
    # Qwen анализирует И текст, И изображение!
    prompt = f"""
    Пользователь сказал: "{raw_text}"
    
    На скриншоте показан интерфейс.
    
    Проанализируй:
    1. Что пользователь нажал? (кнопка, поле, меню?)
    2. Какой элемент интерфейса был затронут?
    3. Напиши чёткую инструкцию с учётом того что видишь на экране
    
    Формат: "Нажмите [ЭЛЕМЕНТ] чтобы [ДЕЙСТВИЕ]"
    """
    
    normalized = await llm.analyze_image_and_text(
        image=screenshot,
        text=prompt
    )
    # Результат: "Нажмите кнопку «Сохранить» чтобы сохранить документ"
    
    normalized_steps.append({
        "step_number": len(normalized_steps) + 1,
        "raw_speech": raw_text,
        "normalized_text": normalized,
        "start": segment['start'],
        "end": segment['end']
    })
```

- При экспорте
- При создании Shorts (там Chatterbox)

```python
def ai_transcription(self, guide_id: int, audio_key: str, language: str = "ru"):
```

```python
# workers/ai_runner.py

def run_ai_subprocess(task_type, task_id, payload):
    """
    Запуск AI задачи в изолированном subprocess.
    
    Преимущества:
    - Краш subprocess не ломает воркер
    - Нет блокировки основного процесса
    - Легко мониторить прогресс
    """
    
    # Создаём временные файлы
    input_file = f"/tmp/input_{task_id}.json"
    output_file = f"/tmp/output_{task_id}.json"
    
    # Записываем входные данные
    with open(input_file, "w") as f:
        json.dump({"task_type": task_type, "payload": payload}, f)
    
    # Запускаем subprocess
    cmd = [sys.executable, "workers/ai_runner.py", 
           "--input", input_file, 
           "--output", output_file]
    
    process = subprocess.Popen(cmd, stdout=PIPE, stderr=PIPE)
    stdout, stderr = process.communicate(timeout=3600)
    
    # Читаем результат
    with open(output_file, "r") as f:
        result = json.load(f)
    
    return result
```

## ⚠️ Возможные проблемы и решения

### Проблема 1: Долгая обработка на CPU

**Симптомы:** Qwen обрабатывает 10+ минут

**Решение:**
- Уменьшить количество потоков: `LLM_THREADS=4`
- Использовать меньшую модель: `WHISPER_MODEL_SIZE=tiny`
- На сервере включить GPU

### Проблема 2: Microphone access denied

**Симптомы:** Расширение не может записать аудио

**Решение:**
- Проверить разрешения браузера
- Явно разрешить доступ к микрофону
- Продолжить запись без аудио (скриншоты всё равно работают)

### Проблема 3: Whisper не распознаёт русский

**Симптомы:** Пустой текст или английский вместо русского

**Решение:**
- Явно указать язык: `language="ru"`
- Проверить качество аудио (шум микрофона)
- Использовать `model="medium"` или `"large"`

### Проблема 4: Qwen выдаёт мусор

**Симптомы:** Нормализация ломает смысл

**Решение:**
- Уменьшить `temperature=0.3` (более детерминировано)
- Улучшить системный промпт
- Добавить примеры в prompt
